HOW TO PUSH A NEW REPO TO GITHUB FOR THE FIRST TIME USING SSH
═══════════════════════════════════════════════════════════════

STEP 1: SET UP SSH KEY (If you haven't already)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1.1 Generate SSH Key:
$ ssh-keygen -t ed25519 -C "your_email@example.com"
    (Press Enter for default location: ~/.ssh/id_ed25519)
    (Enter passphrase if desired, or press Enter to skip)

1.2 Start SSH Agent (macOS/Linux):
$ eval "$(ssh-agent -s)"

1.3 Add SSH Key to Agent:
$ ssh-add ~/.ssh/id_ed25519

1.4 Copy SSH Public Key:
$ cat ~/.ssh/id_ed25519.pub
    (Copy the entire output)

1.5 Add Key to GitHub:
   - Go to: https://github.com/settings/keys
   - Click "New SSH key"
   - Title: "My MacBook" (or your device name)
   - Key type: "Authentication Key"
   - Paste your public key
   - Click "Add SSH key"

1.6 Test SSH Connection:
$ ssh -T git@github.com
    Expected output: "Hi username! You've successfully authenticated..."


STEP 2: CREATE A NEW REPOSITORY ON GITHUB
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

2.1 Go to https://github.com/new

2.2 Fill in:
   - Repository name: "my-project" (example)
   - Description: "Optional description"
   - Visibility: Public or Private
   - DO NOT initialize with README, .gitignore, or license
     (We'll do this locally)

2.3 Click "Create repository"

2.4 Copy SSH URL:
   - You'll see two URL options (HTTPS and SSH)
   - Select SSH tab
   - Copy the URL (should look like: git@github.com:username/my-project.git)


STEP 3: INITIALIZE YOUR LOCAL REPOSITORY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

3.1 Navigate to Your Project Directory:
$ cd /path/to/your/project

3.2 Initialize Git:
$ git init

3.3 Add Files to Staging Area:
$ git add .
    (Or: git add filename.txt  [for specific files])

3.4 Create Initial Commit:
$ git commit -m "Initial commit"


STEP 4: ADD REMOTE AND PUSH TO GITHUB
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

4.1 Add Remote Repository:
$ git remote add origin git@github.com:username/my-project.git
    (Replace 'username' and 'my-project' with your values)

4.2 Verify Remote is Added:
$ git remote -v
    Expected output:
    origin  git@github.com:username/my-project.git (fetch)
    origin  git@github.com:username/my-project.git (push)

4.3 Rename Branch to 'main' (Optional but recommended):
$ git branch -M main

4.4 Push to GitHub:
$ git push -u origin main
    (-u sets upstream, so future 'git push' works automatically)

4.5 Verify Push Success:
    - Go to https://github.com/username/my-project
    - You should see your files and commit history


═══════════════════════════════════════════════════════════════

QUICK REFERENCE COMMANDS (ALL STEPS COMBINED):
═══════════════════════════════════════════════════════════════

$ ssh-keygen -t ed25519 -C "your_email@example.com"
$ eval "$(ssh-agent -s)"
$ ssh-add ~/.ssh/id_ed25519
$ ssh -T git@github.com

$ cd /path/to/your/project
$ git init
$ git add .
$ git commit -m "Initial commit"
$ git remote add origin git@github.com:username/my-project.git
$ git branch -M main
$ git push -u origin main

TROUBLESHOOTING:
═══════════════════════════════════════════════════════════════

❌ "Permission denied (publickey)"
✓ Solution:
  - Verify SSH key was added to GitHub: https://github.com/settings/keys
  - Verify SSH agent is running: eval "$(ssh-agent -s)"
  - Verify key is added to agent: ssh-add ~/.ssh/id_ed25519
  - Test connection: ssh -T git@github.com

❌ "Could not read from remote repository"
✓ Solution:
  - Check SSH URL format: git@github.com:username/repo.git (NOT https)
  - Verify remote: git remote -v
  - Remove wrong remote: git remote remove origin
  - Add correct remote: git remote add origin git@github.com:username/repo.git

❌ "fatal: not a git repository"
✓ Solution:
  - Navigate to correct directory
  - Run: git init

❌ "nothing to commit"
✓ Solution:
  - Add files: git add .
  - Or: git add filename

❌ "Please set your email and name"
✓ Solution:
  - Set globally:
    $ git config --global user.name "Your Name"
    $ git config --global user.email "your_email@example.com"
  - Or locally (in that repo):
    $ git config user.name "Your Name"
    $ git config user.email "your_email@example.com"


═══════════════════════════════════════════════════════════════

IMPORTANT NOTES:
═══════════════════════════════════════════════════════════════

✓ SSH vs HTTPS:
  - SSH: More secure, no password needed (git@github.com:...)
  - HTTPS: Requires authentication each time (https://github.com/...)

✓ -u Flag (upstream):
  - Sets up tracking between local and remote branch
  - Future pushes: just type "git push" (no need for origin main)

✓ Branch Names:
  - GitHub changed default from "master" to "main"
  - Use: git branch -M main (to rename)

✓ First Time Setup:
  - Configure name and email once
  - SSH key setup is one-time per computer
  - After that, all repos use the same SSH key

✓ .gitignore:
  - Create .gitignore file BEFORE git add .
  - Prevents committing sensitive files (passwords, env, node_modules)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  4. Choose: GitHub (for web browser authentication)
  5. Authorize in browser

  gh repo create repo-name --source=. --remote=origin --push

Options explained:
  --source=.          Use current directory as source
  --remote=origin     Set remote as 'origin'
  --push              Automatically push to GitHub

Interactive flags (optional):
  --public            Make repo public (default: private)

# DETAILED REPORT: IRAN-US CONFLICT

## INTRODUCTION
The Iran-United States conflict represents one of the most complex and enduring geopolitical rivalries of the modern era. While the two nations have never formally declared war, they have engaged in decades of proxy conflicts, economic warfare, and diplomatic standoffs that have shaped Middle Eastern politics and global security. The relationship has evolved from cordial allies in the early 20th century to bitter adversaries following the 1979 Islamic Revolution, creating a persistent state of tension that continues to influence international relations today.

## MAIN SECTION

**Paragraph 1: Historical Foundations and the 1953 Coup**
The roots of the Iran-US conflict trace back to the 1953 CIA-backed coup that overthrew Iran's democratically elected Prime Minister, Mohammad Mosaddegh. Mosaddegh had nationalized Iran's oil industry, threatening British and American economic interests. The coup reinstated Shah Mohammad Reza Pahlavi, who ruled with US support until 1979. This intervention created deep-seated resentment among Iranians and established a pattern of American interference that would fuel anti-American sentiment for generations. The Shah's regime, while modernizing Iran, was also repressive and dependent on US support, setting the stage for revolutionary backlash.

**Paragraph 2: The 1979 Revolution and Hostage Crisis**
The 1979 Islamic Revolution fundamentally transformed Iran-US relations, turning former allies into bitter enemies. The revolution overthrew the US-backed Shah and established an Islamic Republic under Ayatollah Khomeini. The subsequent 444-day hostage crisis, where Iranian students seized the US Embassy in Tehran and held 52 Americans captive, cemented the adversarial relationship. This event became a defining trauma in American foreign policy and established Iran as a revolutionary state opposed to Western influence in the region. The hostage crisis led to severed diplomatic relations, economic sanctions, and decades of mutual distrust.

**Paragraph 3: Nuclear Program and the 2015 JCPOA**
Iran's nuclear program became the central point of contention in the 21st century, with the US and its allies accusing Iran of pursuing nuclear weapons capability. After years of escalating sanctions and diplomatic pressure, the 2015 Joint Comprehensive Plan of Action (JCPOA) was negotiated between Iran and world powers. The agreement limited Iran's nuclear activities in exchange for sanctions relief and was hailed as a diplomatic breakthrough. However, the deal faced significant opposition in both countries, with critics arguing it didn't adequately address Iran's regional activities or long-term nuclear ambitions.

**Paragraph 4: Trump Administration and Maximum Pressure**
President Donald Trump's withdrawal from the JCPOA in 2018 marked a dramatic escalation in tensions. The administration implemented a "maximum pressure" campaign of unprecedented sanctions aimed at crippling Iran's economy and forcing concessions. This policy culminated in the January 2020 assassination of Qasem Soleimani, commander of Iran's elite Quds Force, in a US drone strike. The killing brought the two countries to the brink of direct military conflict, with Iran retaliating by launching missile strikes on US bases in Iraq. These events demonstrated how quickly the conflict could escalate despite both sides' stated desire to avoid all-out war.

**Paragraph 5: Current Dynamics and Regional Proxy Conflicts**
The Iran-US conflict has evolved into a complex web of regional proxy wars and strategic competition. Iran supports various militant groups across the Middle East, including Hezbollah in Lebanon, Houthi rebels in Yemen, and Shia militias in Iraq and Syria. The US, in turn, maintains military presence in the region and supports Iran's regional rivals. Recent negotiations in 2025-2026 have attempted to address both nuclear issues and regional security concerns, but fundamental disagreements persist. The conflict continues to influence global oil markets, regional stability, and great power competition between the US, Russia, and China.

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("LocalNotes")

@mcp.tool()
def add_note_to_file(content: str) -> str:
    """
    Appends the given content to the user's local notes.
    Args:
        content: The text content to append.
    """

    filename = 'notes.txt'

    try:
        with open(filename, "a", encoding="utf-8") as f:
            f.write(content + "\n")
        return f"Content appended to {filename}."
    except Exception as e:
        return f"Error appending to file {filename}: {e}"
    

@mcp.tool()
def read_notes() -> str:
    """
    Reads and returns the contents of the user's local notes.
    """
    filename = 'notes.txt'

    try:
        with open(filename, "r", encoding="utf-8") as f:
            notes = f.read()
        return notes if notes else "No notes found."
    except FileNotFoundError:
        return "No notes file found."
    except Exception as e:
        return f"Error reading file {filename}: {e}"
    
mcp.run()
